# 前端启动说明

## 项目说明

本目录为医院预约挂号系统前端，技术栈为：

- `Vue 3`
- `Vite`
- `Vue Router`
- `Element Plus`

## 运行环境

启动前请确认本机已安装：

- `Node.js 18+`
- `npm 9+`

可使用以下命令检查：

```powershell
node -v
npm -v
```

## 安装依赖

先进入前端目录：

```powershell
cd 克隆路径\hospital-appointment-hub\frontend
```

安装依赖：

```powershell
npm install
```

## 启动开发环境

执行：

```powershell
npm run dev
```

启动成功后，默认访问地址为：

- `http://localhost:5173`

## 构建生产包

如果需要打包前端：

```powershell
npm run build
```

构建产物会生成在：

- `frontend/dist`

## 本地代理配置

前端当前通过 Vite 代理后端接口，请求前缀为 `/api`。

代理配置文件：

- `frontend/vite.config.js`

当前代理目标地址为：

- `http://localhost:8080`

也就是说：

- 前端访问 `/api/**`
- 会被自动转发到后端 `http://localhost:8080/api/**`

## 启动前的联调准备

启动前端前，请先确保以下服务已经准备好：

- 后端服务已启动
- MySQL 已启动
- Redis 已启动

如果后端没有启动，前端页面虽然可以打开，但接口请求会失败。

## 常见问题

### 1. 前端能打开，但接口报错

请检查：

- 后端是否已经启动
- 后端端口是否为 `8080`
- 数据库和 Redis 是否正常


### 2. 端口 5173 被占用

如果本机 `5173` 被占用，可以修改：

- `frontend/vite.config.js`

例如改成：

```js
server: {
  port: 5174
}
```

## 补充说明

- 前端 token 保存在浏览器本地存储中
- 业务请求会自动携带 `Bearer` 鉴权头
- 如果修改了前端代码，开发模式下通常会自动刷新
- 如果遇到样式或逻辑未更新，可以手动刷新浏览器
