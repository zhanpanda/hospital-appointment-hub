## 数据库初始化

### 1. 启动 MySQL

确保 MySQL 服务已启动。

### 2. 创建数据库

```sql
CREATE DATABASE IF NOT EXISTS `hospital-appointment-hub`
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_0900_ai_ci;
```

### 3. 导入初始化脚本

```bash
# 方式一：命令行导入
mysql -u root -p hospital-appointment-hub < backend/src/main/resources/sql/init.sql

# 方式二：用 Navicat/DBeaver 等工具打开 init.sql 执行
```

### 4. 导入测试数据（可选）

数据库表创建完成后，还需要导入之前准备的测试数据：

| 数据               | SQL 来源                           |
| ------------------ | ---------------------------------- |
| 科室数据 (20条)    | 之前生成的 INSERT INTO departments |
| 医生数据 (47条)    | 之前生成的 INSERT INTO doctors     |
| 排班数据 (未来7天) | 之前生成的 INSERT INTO schedules   |

### 5. 修改配置（如需要）

打开 `backend/src/main/resources/application-dev.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/hospital-appointment-hub
    username: root
    password: 1234        # ← 改成你的 MySQL 密码

  data:
    redis:
      host: localhost
      port: 6379
      password: 1234      # ← 改成你的 Redis 密码（如无密码则删除此行）
```

## 后端启动

### 1. 进入后端目录

```bash
cd backend
```

### 2. 启动方式

**方式一：Maven 命令**

```bash
mvn spring-boot:run
```

**方式二：IDE 直接启动**

在 IntelliJ IDEA 或 Eclipse 中：

1. 找到启动类 `HospitalAppointmentHubApplication`
2. 右键 → Run

**方式三：打包运行**

```bash
mvn clean package -DskipTests
java -jar target/hospital-appointment-hub-*.jar
```

### 3. 验证后端启动

启动成功后，控制台会显示：

```
Started HospitalAppointmentHubApplication in x.xxx seconds
```

访问测试接口：

```bash
curl http://localhost:8080/api/test
```

或者浏览器打开 `http://localhost:8080`

## 常见问题

### ❌ MySQL 连接失败

```
CommunicationsException: Communications link failure
```

**解决**：
- 确认 MySQL 已启动
- 检查 `application-dev.yml` 中用户名密码
- 确认数据库已创建

### ❌ Redis 连接失败

```
Cannot connect to Redis
```

**解决**：
- 确认 Redis 已启动：`redis-cli ping`
- 检查 Redis 密码配置
- 如无密码，删除配置文件中 `password` 行

### ❌ 后端端口 8080 被占用

**解决**：修改 `application-dev.yml`：

```yaml
server:
  port: 8081
```

同时修改前端 `vite.config.js` 代理目标：

```js
proxy: {
  '/api': {
    target: 'http://localhost:8081',  // ← 改这里
  }
}
```

### ❌ 前端端口 5173 被占用

**解决**：修改 `frontend/vite.config.js`：

```js
server: {
  port: 5174  // ← 改成其他端口
}
```

### ❌ 前端能打开但接口 404

**解决**：
- 确认后端已启动
- 确认后端端口与 `vite.config.js` 代理目标一致
- 重启前端 `npm run dev`

### ❌ 登录后接口 401

**解决**：
- 检查 Redis 是否正常
- 清除浏览器 localStorage 重新登录

### ❌ npm install 失败

**解决**：
```bash
# 清除缓存重试
npm cache clean --force
npm install

# 或使用国内镜像
npm install --registry=https://registry.npmmirror.com
```
